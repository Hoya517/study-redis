# Redis 실습 - 블로그 조회수 관리

------------------------------------------------------------------------

# 1️⃣ 전체 조회수 관리

## 📌 요구사항

-   URL 형식: `/articles/{id}`
-   로그인 여부와 관계없이 새로고침 시마다 조회수 증가
-   Redis를 이용하여 조회수 저장

------------------------------------------------------------------------

## ✅ 데이터 타입 선정

**String 타입 사용**

### 이유

-   단순 카운트 증가 목적
-   Redis의 원자적 연산 `INCR` 사용 가능
-   동시성 안전

------------------------------------------------------------------------

## 📌 기본 조회수 증가

### Key 설계

    articles:{id}

### 페이지 접속 시 실행 명령

``` bash
INCR articles:1
INCR articles:2
INCR articles:3
```

------------------------------------------------------------------------

## 📌 오늘의 조회수 따로 관리

``` bash
INCR articles:1:today
```

자정에 날짜 키로 변경:

``` bash
RENAME articles:1:today articles:1:2026-02-27
```

### 💡 권장 방식

날짜를 처음부터 키에 포함:

``` bash
INCR articles:1:2026-02-27
```

# 2️⃣ 로그인 사용자 조회수 및 인기글 관리

## 📌 요구사항

-   URL 형식: `/articles/{id}`
-   로그인 사용자 계정은 영문
-   로그인한 사용자 기준 조회수 관리
-   가장 조회수가 높은 글 확인

------------------------------------------------------------------------

## ✅ 데이터 타입 선정

목적                      데이터 타입
  ------------------------- -------------
로그인 사용자 중복 방지   Set
인기글 랭킹 관리          Sorted Set

------------------------------------------------------------------------

## 📌 로그인 사용자 조회 관리 (Set)

``` bash
SADD articles:1 alex
SADD articles:1 brad
SADD articles:1 chad

SCARD articles:1
```

다른 글 예시:

``` bash
SADD articles:2 alex
SADD articles:2 chad

SCARD articles:2
```

------------------------------------------------------------------------

## 📌 조건부 로직

`SADD` 결과값:

-   0 → 이미 존재 (증가 스킵)
-   1 → 새로운 사용자 (랭킹 증가)

``` bash
ZINCRBY articles:ranks 1 articles:1
ZINCRBY articles:ranks 1 articles:2
```

------------------------------------------------------------------------

## 📌 가장 조회수 높은 글 조회

``` bash
ZREVRANGE articles:ranks 0 0
```

또는

``` bash
ZRANGE articles:ranks 0 0 REV
```

---

# 📊 정리

| 기능 | Redis 타입 | 사용 명령어 |
|------|------------|-------------|
| 전체 조회수 | String | INCR |
| 일별 조회수 | String | INCR |
| 로그인 사용자 관리 | Set | SADD, SCARD |
| 인기글 랭킹 | Sorted Set | ZINCRBY, ZREVRANGE |

---

# 💡 실무 관점 확장

-   TTL 설정 가능 (`EXPIRE`)
-   트래픽 증가 시 Redis 카운터는 매우 효율적
-   랭킹 + 조회수 조합 시 Sorted Set 필수
