# Redis Leaderboard Practice (Sorted Set)

Spring Boot와 Redis의 Sorted Set을 이용하여 실시간 인기 상품 리더보드
기능을 구현한 실습 프로젝트입니다.

------------------------------------------------------------------------

## 📌 학습 목표

-   리더보드(Leaderboard) 기능의 개념 이해
-   관계형 데이터베이스로 구현 시의 한계 이해
-   Redis Sorted Set의 동작 원리 이해
-   Spring Data Redis + ZSetOperations 활용

------------------------------------------------------------------------

## 🛠 Tech Stack

-   Java 17
-   Spring Boot 3.x
-   Spring Data Redis
-   Redis Stack (Docker)

------------------------------------------------------------------------

## 🚀 Redis 실행

``` bash
docker compose up -d
```

Redis 기본 포트: localhost:6379

------------------------------------------------------------------------

## 1️⃣ 리더보드란?

리더보드는 점수 기반으로 순위를 정렬하여 보여주는 기능입니다.

예시:

-   게임 점수 랭킹
-   실시간 검색어
-   이커머스 인기 상품

------------------------------------------------------------------------

## 2️⃣ RDB로 구현할 경우 문제점

예를 들어 인기 상품을 "가장 많이 구매된 상품"으로 정의하면 다음과 같은
쿼리가 필요합니다.

``` sql
SELECT i.id, SUM(o.count)
FROM item i
INNER JOIN orders o ON i.id = o.item_id
GROUP BY i.id
ORDER BY SUM(o.count) DESC
LIMIT 10;
```

문제점:

-   Join + 집계 함수 필요
-   빈번한 UPDATE 발생
-   정렬 비용 증가
-   트래픽 증가 시 성능 저하 가능성

------------------------------------------------------------------------

## 3️⃣ Redis Sorted Set 사용

Redis Sorted Set 특징:

-   score 기반 자동 정렬
-   삽입 시간복잡도: O(log N)
-   조회 시간복잡도: O(log N + M)

### 주요 명령어

-   ZADD
-   ZINCRBY
-   ZRANGE
-   ZREVRANGE

------------------------------------------------------------------------

## 4️⃣ 구현 방식

### 상품 구매 시

``` java
zSetOperations.incrementScore("leaderboard", itemId, 1);
```

-   상품 구매 발생 시 score 증가
-   score = 총 판매 수량

### 인기 상품 조회

``` java
zSetOperations.reverseRange("leaderboard", 0, 9);
```

-   score 높은 순으로 정렬
-   상위 10개 반환

------------------------------------------------------------------------

## 📊 RDB vs Redis 비교

| 구분 | RDB 방식 | Redis Sorted Set |
|---|---|---|
| 정렬 처리 | ORDER BY 필요 | 자동 정렬 |
| 성능 | 집계/정렬 비용 발생 | 인메모리 기반 빠름 |
| 구현 난이도 | 상대적으로 복잡 | 단순 |
| 확장성 | 부하 증가 가능 | 고성능 처리 가능 |

------------------------------------------------------------------------

## 🎯 실무 관점 정리

-   실시간 랭킹 기능은 Sorted Set이 최적
-   RDB는 트랜잭션 관리에 집중
-   Redis는 실시간 집계 및 캐시 역할로 활용

Redis의 데이터 타입을 적절히 선택하면 특정 기능 구현 난이도를 크게 낮출
수 있습니다.
